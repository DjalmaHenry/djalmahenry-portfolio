# Guia de Conteudo Visual

Este arquivo explica o que inserir em cada espaco visual reservado no portfolio.

## Visual 02 - Arquitetura, stack ou bastidores

- Objetivo: mostrar profundidade tecnica sem poluir a leitura.
- O que inserir: diagrama simples de arquitetura, collage de tecnologias, fluxo de microsservicos ou screenshot de observabilidade.
- Formato ideal: arte mais tecnica, com poucos elementos e boa hierarquia.
- Prioridade: usar algo que comunique organizacao, escala e engenharia.

## Visual 03 - Timeline, dashboard ou contexto de operacao

- Objetivo: apoiar a secao de experiencia com um elemento visual que traduza maturidade profissional.
- O que inserir: dashboard real ou conceitual, timeline visual da carreira, print de produto interno ou painel de operacao.
- Formato ideal: imagem horizontal com leitura clara mesmo em tamanho medio.
- Prioridade: evidenciar contexto profissional e impacto de negocio.

## Visual 04 - Mockup do produto Shoryu

- Objetivo: dar vida ao projeto com uma apresentacao visual mais forte.
- O que inserir: mockup da plataforma, chaveamento, tela de partidas, fluxo de ingressos ou composicao com multiplas telas.
- Formato ideal: conjunto desktop + mobile ou painel principal do sistema.
- Prioridade: destacar organizacao, usabilidade e identidade do produto.

## Visual 05 - Tela comercial + dashboard Visuh

- Objetivo: mostrar o contraste entre experiencia comercial e operacao.
- O que inserir: vitrine digital, configuracao de produto, dashboard de estoque ou metricas de vendas.
- Formato ideal: comparativo entre tela publica e tela administrativa.
- Prioridade: evidenciar valor comercial e clareza de interface.

## Recomendacoes gerais

- Use imagens com boa resolucao e enquadramento consistente.
- Evite prints com excesso de texto pequeno ou muitos elementos simultaneos.
- Sempre que possivel, prefira mockups bem compostos em vez de screenshots crus.
- Mantenha a paleta visual alinhada ao site para preservar unidade.
